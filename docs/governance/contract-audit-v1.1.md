## Binding execution notes (v1.1)

1. **Dispatch transport (ADR-006):** `services/capture-win/crates/dispatch` posts Zod/serde-validated JSON to the `assist` Edge Function over HTTPS with connection reuse, exponential backoff + jitter, blake3 payload checksum, and a bounded drop-oldest offline queue. gRPC/tonic is deleted from scope; nothing downstream consumed it.
2. **Contract source (ADR-007):** `packages/contracts` (Zod) is canonical. `scripts/sync-edge.mjs` copies the schema module into `supabase/functions/_shared/contracts.ts`; CI fails on hash drift. Rust `dispatch` crate defines serde structs verified against the same golden fixtures.
3. **Title registry:** 16 grounded titles seeded (6 Wave-1, 10 registry-only). Gap of 4 logged as UNCERTAIN, not filled.
4. **Model routing:** alias-driven; escalation threshold `confidence < 0.55` and dense-text ROI flag as contracted; circuit breaker flips to fallback alias when the rolling mean per-assist cost exceeds `COST_BREAKER_USD_MICROS` (default 500).
5. **Verification ledger:** `docs/evidence/` distinguishes RAN (verbatim output) / RUNBOOK (committed suite, needs Supabase or Windows) for every VERIFY clause. GO requires the runbook items executed by JR before release — unchanged from v1.0 §4/WP-7.
